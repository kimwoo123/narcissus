# Claude JSONL 뷰어

`~/.claude/projects` 아래에 쌓이는 Claude Code 세션 기록(`.jsonl`)을 브라우저에서 읽기 좋게 보여주는 로컬 뷰어입니다. 외부 런타임 없이 도는 **Go 단일 바이너리**이고, HTML/JS/CSS는 바이너리 안에 내장됩니다.

```
┌─────────────┬──────────────┬──────────────┐
│ 프로젝트 ▾   │  나        💬 │  나        💬 │
│             │   ┌─────────┐ │   ┌────────┐ │
│ 🏷 세션 A    │   │ 파란말풍선│ │   │…      │ │
│   세션 B    │ Claude       │ Claude       │
│   세션 C    │ ┌──────────┐ │ ┌──────────┐ │
│             │ │초록말풍선  │ │ │…         │ │
└─────────────┴──────────────┴──────────────┘
   사이드바        패널 1          패널 2 (최대 4)
```

## 기능

- **세션 탐색** — 프로젝트 선택 → 세션 목록. 제목은 Claude가 붙인 `ai-title`(가장 최근 것)을 쓰고, 없으면 첫 사용자 메시지를 씁니다.
- **분할뷰** — 세로 컬럼으로 최대 4개 세션을 나란히. 클릭은 활성 패널 교체, **⌘/Ctrl+클릭**은 새 패널.
- **대화 표시** — 내 메시지는 우측 파란 말풍선, Claude는 좌측 초록 말풍선. 최신 메시지가 맨 위.
- **도구 호출 묶음** — 도구 호출과 그 결과를 한 카드로 짝지어 표시.
- **마크다운 렌더링** — Claude 답변의 제목·굵게·코드·표·리스트를 렌더링 (내 메시지·도구 로그는 원문 유지).
- **실시간 갱신** — 파일이 바뀌면 화면이 즉시 따라옵니다 (진행 중인 세션을 `tail -f`처럼 관찰).
- **사용자 라벨** — 세션에 내 이름표를 달 수 있습니다. 원본 파일은 건드리지 않습니다.

## 요구사항

- Go 1.24+
- macOS / Linux / Windows (파일 감시는 fsnotify가 OS별 커널 API로 처리)

## 설치 · 실행

```bash
cd jsonl-viewer-go
go build -o jsonl-viewer    # 단일 바이너리 생성
./jsonl-viewer              # 실행 → 브라우저가 http://localhost:8765 자동 오픈
```

빌드 없이 바로 실행하려면:

```bash
go run .
```

종료는 `Ctrl+C`. 어디서나 실행하고 싶으면 바이너리를 PATH로 복사하세요:

```bash
cp jsonl-viewer /usr/local/bin/
```

## 사용법

| 동작 | 설명 |
|---|---|
| 세션 **클릭** | 활성 패널에서 열기 (패널이 없으면 새로 생성) |
| **⌘/Ctrl + 클릭** | 새 패널로 열기 (최대 4) |
| 패널 클릭 | 그 패널을 활성화 (파란 헤더) — 다음 일반 클릭의 대상 |
| 패널 헤더 **×** | 패널 닫기 |
| 패널 제목 클릭 | 라벨 편집 (Enter 저장 / 비우고 Enter = 라벨 제거 / Esc 취소) |

라벨은 `~/.claude/jsonl_viewer_labels.json`에 `"프로젝트/파일": "라벨"` 형태로 저장됩니다. `ai-title`과 무관하게 항상 우선 표시되며, 라벨이 달린 세션은 사이드바에 🏷로 표시됩니다.

## 코드 구조

| 파일 | 역할 |
|---|---|
| `main.go` | 진입점 — 전역 설정, HTTP 라우팅, 서버 기동 |
| `parse.go` | JSONL 파싱 — 제목 추출, 메시지 블록(text/thinking/tool/result) 펼치기 |
| `cache.go` | 증분 파싱 캐시 — 바뀐 부분만 다시 파싱 |
| `api.go` | 응답 조립 — 목록/세션 렌더링, 경로 검증 |
| `markdown.go` | Claude 텍스트의 마크다운 → HTML 변환 (goldmark) |
| `labels.go` | 사용자 라벨 저장소 |
| `watch.go` | 변경 감지 — fsnotify 감시 + SSE 허브 |
| `index.html` | 프론트엔드 (go:embed로 바이너리에 내장) |

## 동작 원리

**실시간 갱신 (fsnotify + SSE).** 서버가 `~/.claude/projects`를 커널 파일 이벤트로 감시하다가, 변경이 생기면 SSE(`/api/events`)로 브라우저에 밀어줍니다. 브라우저는 폴링하지 않고 받기만 합니다.
- 세션 파일 *수정* → 그 파일을 보고 있는 모든 패널을 갱신
- 세션 파일 *생성·삭제* → 사이드바 목록 갱신

**증분 파싱 캐시.** JSONL은 거의 append-only라, 파일이 커지면 **추가된 꼬리만** 파싱하고 나머지는 캐시를 재사용합니다. 진행 중인 세션을 갱신할 때마다 수 MB를 통째로 재파싱하지 않습니다.

**마크다운 안전성.** goldmark를 `unsafe` 옵션 없이 사용하므로, 본문에 섞인 원시 HTML(`<script>` 등)은 렌더링되지 않습니다. 변환 결과를 그대로 화면에 넣어도 XSS 위험이 없습니다.

## 데이터 · 보안

- 서버는 `127.0.0.1`에만 바인딩됩니다 (로컬 전용).
- 모든 경로는 `~/.claude/projects` 안으로 제한됩니다 (디렉터리 탈출 차단).
- 세션 원본 파일은 **읽기 전용**으로만 다룹니다. 쓰기는 별도 라벨 파일에만 일어납니다.

## API

| 엔드포인트 | 설명 |
|---|---|
| `GET /api/projects` | 프로젝트 목록 |
| `GET /api/sessions?project=` | 해당 프로젝트의 세션 목록 |
| `GET /api/session?project=&file=` | 세션 대화 내용 (마크다운 렌더 포함) |
| `POST /api/label` | 라벨 설정/제거 `{project, file, label}` |
| `GET /api/events` | SSE 변경 스트림 |

## 의존성

- [fsnotify](https://github.com/fsnotify/fsnotify) — 크로스플랫폼 파일 감시
- [goldmark](https://github.com/yuin/goldmark) — 마크다운 → HTML (GFM)
